Fresh Installation Instructions

Important

When you run the bot for the very first time, it will check if a certain column exists in your 'posts' table. If it doesn't, then it will automatically add the column. However, it may take sometime to do that if your database is huge. So I recommend you take your board offline at the first run.
Also, if you have used my previous bots for IPB2, then UNDO all the database edits before proceeding.

Step 1: Upload Required Files

File Upload To
/upload/admin/applications/forums/tasks/link_bot.php /admin/applications/forums/tasks/
/upload/admin/applications/forums/tasks/mscookie /admin/applications/forums/tasks/

Step 2: Import Hook

Open the Admin CP > System Tab > Manage Hooks. Scroll down to the 'Install a New Hook' section, browse and select the hook.xml file included with these instructions, then click the Install button.

Step 3: Configure Bot and Task

Open the Admin CP > System Tab > Tools & Settings > System Settings. Click on the 'Forums' tab and then click 'Link Bot [nihalz]'. You can configure various settings for the bot here.
Open the Admin CP > System Tab > System > System Scheduler. Click on the 'Forums' tab. You will see a task named 'Link Bot [nihalz]'. You can configure the frequency of the bot here and also disable it.

Notes

The bot logs it's actions at certain events along with some statistics. You can view them in the 'Logs' section in Admin CP > System Tab > Tools & Settings > Logs > Scheduler Logs

If the bot scans a post which is not the first post of the topic, and it finds dead links in it, it will 'Unapprove' the post. On the other hand, if it is the first post of the topic, it will move the topic to the specified forum; provided 'Auto Deletion' is enabled.

The bot checks RS links from a post in a single batch. From my testing, I have concluded that if you have more than 140 RS links in a single post, Rapidshare's servers will reject the request and the links won't be checked.
